Chapter 1 - Variables

A very basic first chapter. Important to learn some of the basic JS syntax.

1. Console.log

JavaScript's version of the print command. Mostly used for debugging.


2. Comments

Comments on JavaScript are declared with '//' for single line comments and /* */ for multi-line comments.

It is good practice to leave single-line comments at the end of code lines.


3. Iterative Language

Similarly to Python, JavaScript is an iterative language, meaning it runs from top to bottom.


4. Semicolons

Similarly to Java, JS should have semicolons at the end of each command. Unlike Java however, JS will still run the program, however it is good practice to always include semicolons in the correct places.


5. Variables

To initialize a var in JS, the following format is used: "let varName= value".

Note: Coddy stores booleans with all lower case. However is it good practice to use capitalized booleans? 'True'!


6. Naming Conventions

Variables are case-sensitive in JS, and like other languages, cannot begin with a number.

The common practice for naming variables in JS is camelCase.


7. Empty Variables

Like other programming languages, JS allows us to declare variables without values. We can do this by simply typing 'let myVariable'. This can be assigned a value later (or not).


8. Constants

Declared with 'const' in the place of 'let': constants are simply variables that cannot be changed after declaration.

Chapter 2 - Variables

The second chapter.

1. Basic Arithmetic Operators

Operators are used to perform operations on values. The most basic operatics are:

Addition (+)

Subtraction (-)

Multiplication (*)

Division (/)


2. Modulo Operator

The modulo operator (%) returns what's left after dividing one number by another. For example, if we did 9 % 2, this would return the output of 1.


3. Arithmetic Shortcuts

Similarly to other languages, JS has shortcuts for self-arithmetic operations. For example instead of doing vat = var + 3, we can simply do var += 3. This is the same for all of the operators in the previous two sections, using (-=), (*=) etc.


4. Comparison Operators

These are used to compare two operands. For example, we can check if two operands are equal (==), not equal (!=), greater than (>=) etc.


5. Strict vs Loose Equality

In JS, there exist two types of equality operators, loose equality (==) and strict equality (===). When i first started writing in JS, I had no idea why three === were used lol. A simple loose (==) equality checks if two values are equal after performing type coersion, whereas strict equality (===) checks if the two values are equal without performing any type conversion. As a result, it is recommended that strict equality is used for most cases to avoid unexpected results.


6. Logical Operators

Logical operators are used to check combinations of comparisons that return true or false. For example, in an if statement, we could use &&, checking if both are true, ||, checking if any is true, or !, seeing if it is false.

We can also rearrange logical operators to give us different outcomes. For example, !(A && B ) is the same as (!A) || (!B).


7. Type Coercion

Type coercion in JS is the automatic conversion of values from one data type to another. This can happen when you call functions like String(), Number() or Boolean() (similarly to python).

Implicit type coercion often happens when the loose equality operator (==) is used, or when operations are performed between different types.
